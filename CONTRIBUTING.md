# Contributing

This repository uses a simple GitHub Flow process to keep team work organized and safe.

## Before You Start

- Check for an existing GitHub issue.
- Create a new issue if the work is not already tracked.
- Create a feature branch from `main` using the branch naming convention in [TEAM_WORKFLOW.md](TEAM_WORKFLOW.md).

## Working on a Task

- Keep your branch focused on one issue.
- Make small, descriptive commits.
- Use conventional commit messages such as `feat: add delay chart` or `fix: handle missing values`.
- Push your branch regularly so teammates can follow progress.

## Pull Request Rules

- Open a pull request when the task is ready for review.
- Link the related issue in the PR description.
- Include testing notes and a short summary of the change.
- Ask at least one teammate to review before merging.

## Merge Rules

- Merge only after approval.
- Keep `main` stable.
- Delete the feature branch after merge.
